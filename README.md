# النظام المحاسبي — مشروع جاهز للبناء (أندرويد + سطح مكتب)

## تحديث: ميزات ذكية جديدة أُضيفت للنظام

- **ترقية VIP تلقائية:** عند تجاوز مشتريات العميل خلال 30 يومًا حدًا معينًا (قابل للتعديل من الإعدادات)، يُصنَّف VIP تلقائيًا ويُفعَّل له خصم تلقائي في فواتيره القادمة.
- **تصنيف مخاطر التأخر + تذكيرات واتساب:** النظام يجهز رسالة تذكير مناسبة (لطيفة قبل الاستحقاق بأسبوع، حازمة يوم الاستحقاق، وتنبيه رسمي عند التأخر) ويأخذك بضغطة واحدة لإرسالها عبر واتساب - راجع تبويب "تذكيرات التحصيل".
- **التنبؤ بالطلب:** تبويب "التنبؤ بالطلب" يحلل مبيعات آخر 3 أشهر ويقترح كميات إعادة الطلب.
- **الجدارة الائتمانية:** النظام يمنع البيع الآجل تلقائيًا للعملاء المتأخرين في السداد (بحدود قابلة للتعديل)، مع إمكانية تجاوز يدوي عند الحاجة.
- **الفواتير الدورية:** تبويب "الفواتير الدورية" لإنشاء اشتراكات متكررة (شهرية/أسبوعية) تُصدر نفسها تلقائيًا.
- **المندوبون والعمولات:** تبويب "المندوبون والعمولات" يربط كل فاتورة بمندوب ويحسب عمولته لحظيًا - أساس نظام "الأداء مقابل الدخل".

**ملاحظة مهمة:** إرسال واتساب يتم بضغطة واحدة (رابط جاهز مسبقًا) وليس إرسالًا صامتًا تلقائيًا بدون تدخل - هذا يتطلب اشتراكًا مدفوعًا بخدمة WhatsApp Business API، وهو خارج نطاق هذا التحديث المجاني.

**نصيحة إذا كنت تتابع من الموبايل:** بما أن حجم الكود زاد بشكل كبير (أكثر من 3000 سطر)، تعديل الملف يدويًا على المتصفح بالموبايل أصبح غير عملي. الطريقة الأسهل الآن: احذف الملفات القديمة من مستودعك على GitHub بالكامل، ثم ارفع محتويات هذا المجلد المضغوط من جديد دفعة واحدة (نفس طريقة "uploading an existing file" بالسحب والإفلات). إن أمكنك الوصول لجهاز كمبيوتر ولو لمرة واحدة لهذه الخطوة، سيكون أسرع وأضمن.

---

هذا مشروع React كامل يحتوي على نظام محاسبي متكامل (فواتير، مخزون، عملاء/موردون، قيود يومية، تقارير مالية). نفس الكود يُبنى مرتين: مرة كتطبيق أندرويد عبر **Capacitor**، ومرة كتطبيق سطح مكتب عبر **Electron**.

> **ملاحظة:** كل الأوامر التالية تُنفَّذ على جهازك أنت (وليس من هنا)، لأنها تحتاج اتصال إنترنت لتحميل الحزم، وبرامج مثل Android Studio للتوقيع والبناء النهائي.

## المتطلبات الأساسية

قبل أي شيء، ثبّت على جهازك:

1. **Node.js** (نسخة 18 أو أحدث) من [nodejs.org](https://nodejs.org)
2. للأندرويد فقط: **Android Studio** (يشمل Android SDK) من [developer.android.com/studio](https://developer.android.com/studio) + **Java JDK 17**
3. لسطح المكتب: لا تحتاج شيئًا إضافيًا (Electron يعمل مباشرة)

تأكد أن Node.js مثبت بكتابة في الطرفية (Terminal):
```
node -v
npm -v
```

## الخطوة 1: تثبيت الحزم (مرة واحدة فقط)

افتح الطرفية داخل مجلد المشروع ونفّذ:

```bash
npm install
```

سيقوم هذا بتحميل React وElectron وCapacitor وكل ما يلزم (قد يأخذ بضع دقائق حسب سرعة الإنترنت).

## الخطوة 2: تجربة النظام في المتصفح أولًا (اختياري لكن مفيد)

```bash
npm run dev
```

سيفتح رابط محلي (مثل `http://localhost:5173`) — افتحه في المتصفح للتأكد أن كل شيء يعمل قبل البناء النهائي.

## 🖥️ بناء تطبيق سطح المكتب (Electron)

### تجربة سريعة أثناء التطوير:
```bash
npm run electron:dev
```
يفتح نافذة تطبيق سطح مكتب حقيقية تعمل بنفس الكود.

### البناء النهائي (يُنتج ملف تثبيت حقيقي):
```bash
npm run electron:build
```

النتيجة تظهر داخل مجلد `release/`:
- على ويندوز: ملف `.exe` (مثبّت NSIS)
- على ماك: ملف `.dmg` (يتطلب تشغيل الأمر من جهاز Mac)
- على لينكس: ملف `.AppImage`

> **مهم:** إلكترون يبني فقط للنظام الذي تشغّل عليه الأمر. يعني إذا أردت `.exe` يجب تشغيل الأمر من جهاز ويندوز، و`.dmg` من جهاز ماك، وهكذا. توجد خدمات سحابية (مثل GitHub Actions) تبني للأنظمة الثلاثة معًا إذا احتجت ذلك لاحقًا.

يمكنك أيضًا بناء نظام معيّن فقط:
```bash
npm run electron:build:win
npm run electron:build:mac
npm run electron:build:linux
```

## 📱 بناء ملف APK بدون تثبيت Android Studio (الطريقة الأسهل)

جهّزت لك ملف `.github/workflows/build-android-apk.yml` يبني تطبيق أندرويد **تلقائيًا في السحابة** عبر GitHub Actions (مجاني تمامًا)، وينتج لك ملف APK جاهز للتثبيت المباشر على أي هاتف — بدون أي برنامج تثبته على جهازك، وبدون نشر على المتجر.

### الخطوات:

1. أنشئ حسابًا مجانيًا على [github.com](https://github.com) إن لم يكن لديك واحد
2. أنشئ مستودعًا (Repository) جديدًا فارغًا (خاص أو عام، حسب رغبتك)
3. ارفع محتويات هذا المشروع كاملة إليه. من داخل مجلد المشروع في الطرفية:
   ```bash
   git init
   git add .
   git commit -m "أول رفع"
   git branch -M main
   git remote add origin https://github.com/USERNAME/REPO_NAME.git
   git push -u origin main
   ```
   (استبدل `USERNAME/REPO_NAME` برابط مستودعك الفعلي)
4. بمجرد الرفع، ستبدأ عملية البناء تلقائيًا. لمتابعتها: افتح المستودع على GitHub → تبويب **Actions** → ستجد عملية بناء تعمل (أو اضغط **Run workflow** لتشغيلها يدويًا)
5. انتظر حتى تكتمل (عادة 3-5 دقائق) — علامة ✅ خضراء تعني النجاح
6. اضغط على عملية البناء المكتملة → في الأسفل ستجد قسم **Artifacts** → حمّل `app-debug-apk` (ملف مضغوط يحتوي على `app-debug.apk`)

### تثبيت الـ APK على الهاتف:
1. انقل ملف `app-debug.apk` إلى هاتف أندرويد (عبر رابط تحميل، بلوتوث، أو كابل USB)
2. افتح الملف من الهاتف — سيطلب منك تفعيل "السماح بالتثبيت من مصادر غير معروفة" لمرة واحدة (من الإعدادات)
3. أكمل التثبيت — التطبيق يعمل الآن مباشرة على الهاتف بكل مزاياه

> هذا **APK للتجربة/الاستخدام الشخصي أو التوزيع المباشر** (يُسمى "debug build") — يعمل بشكل كامل تمامًا، لكنه غير موقّع بشهادة نشر رسمية. إذا احتجت لاحقًا رفعه على Google Play، ستحتاج بناء نسخة "release" موقّعة، وهذا موضح في القسم التالي عبر Android Studio.

## 📱 بناء تطبيق أندرويد عبر Android Studio (للتحكم الكامل أو النشر على المتجر)

### أول مرة فقط — إضافة منصة أندرويد للمشروع:
```bash
npm run cap:add:android
```
هذا ينشئ مجلد `android/` كامل (مشروع Android Studio حقيقي).

### كل مرة تعدّل فيها الكود، لتحديث نسخة أندرويد:
```bash
npm run cap:sync
```

### فتح المشروع في Android Studio:
```bash
npm run cap:open:android
```
سيفتح Android Studio تلقائيًا. من داخله:
1. انتظر حتى ينتهي من مزامنة Gradle (أول مرة قد تأخذ عدة دقائق)
2. لتجربة سريعة: اضغط ▶️ (Run) لتشغيله على محاكي أو جهاز حقيقي موصول
3. لإنتاج ملف تثبيت حقيقي: من القائمة اختر
   `Build → Generate Signed Bundle / APK`
   - اختر **APK** للتجربة المباشرة، أو **Android App Bundle (AAB)** وهو المطلوب لرفعه على Google Play
   - أول مرة سيطلب منك إنشاء "Keystore" (ملف توقيع) — احتفظ به في مكان آمن، ستحتاجه في كل تحديث مستقبلي للتطبيق

### للنشر على Google Play:
تحتاج:
- حساب مطوّر Google Play (رسوم اشتراك لمرة واحدة ~25 دولار)
- ملف AAB موقّع (من الخطوة السابقة)
- أيقونة التطبيق + لقطات شاشة + وصف — يمكن تجهيزها من داخل Android Studio أو أدوات تصميم خارجية

## تخصيص اسم التطبيق والأيقونة

- **اسم التطبيق:** غيّر `appName` في `capacitor.config.json` و`productName` في `package.json`
- **معرّف التطبيق (Package ID):** غيّر `appId` في كلا الملفين (يفضّل تنسيق `com.yourcompany.appname`) — **يجب تغييره قبل النشر** إلى معرّف خاص بك
- **الأيقونة:**
  - لأندرويد: استخدم أداة [Capacitor Assets](https://capacitorjs.com/docs/guides/splash-screens-and-icons) لتوليد كل أحجام الأيقونات تلقائيًا من صورة واحدة
  - لسطح المكتب: ضع ملف `icon.png` (أو `.ico` لويندوز، `.icns` لماك) داخل `resources/` وأضف مسارها في حقل `build` بملف `package.json`

## ملاحظة عن حفظ البيانات

النظام يحفظ كل بياناته (الفواتير، العملاء، المخزون...) محليًا على الجهاز نفسه باستخدام `localStorage`، فتبقى البيانات محفوظة بين مرات التشغيل. إن أردت لاحقًا مزامنة البيانات بين عدة أجهزة (موبايل + سطح مكتب لنفس المحل)، ستحتاج خادمًا مركزيًا (قاعدة بيانات سحابية) بدل التخزين المحلي — وهذه خطوة تالية منفصلة يمكن إضافتها عند الحاجة.
